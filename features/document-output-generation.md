# Document Output Generation

## Description

Agents produce documents as task outputs. After task completion, output directories are scanned for generated files, which are registered as documents with `direction="output"`. Supports version tracking for re-runs and common output formats (markdown, JSON, CSV, text, HTML).

## User Story

As a user, I want to see all files my agents generate as task outputs, organized alongside input documents, so I can easily access and manage generated reports, code, and data files.

## Technical Approach

### Output Directory Convention

- Each task gets an output directory: `~/.stagent/outputs/{taskId}/`
- Agent prompt includes the output directory path
- Agent writes files to the directory using standard file operations

### Output Scanning

- After task completion (status → completed), scan output directory
- Register discovered files as documents with `direction="output"`
- Extract metadata: filename, size, MIME type, creation time

### Version Tracking

- Re-running a task with the same output filename increments version
- Version stored in document metadata
- Previous versions remain accessible

### UI Integration

- Task detail shows output documents separately from input attachments
- Output documents browsable in Document Manager (filter by direction)
- Common formats: .md, .json, .csv, .txt, .html

## Scope Boundaries

**Included:**
- Output directory convention and scanning
- Document registration for output files
- Version tracking on re-runs
- Separate display of input vs output documents

**Excluded:**
- Real-time file watching during execution
- Agent-initiated file upload API
- Binary output formats (images, PDFs generated by agents)
- Output file diffing between versions
